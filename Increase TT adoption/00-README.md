Turn the SSP training output into real TT adoption: identify which accounts are ready, audit their platform state, and schedule demos with clients.

## How to use this folder

1. `01-ssp-analysis.md` — Full analysis of SSP responses: tiers, flags, completeness
2. `02-account-audit-tracker.md` — Per-account audit checklist (variables, targets, projects, dictionaries)
3. `03-demo-pipeline.md` — Demo scheduling tracker with SSPs and dates
4. `04-action-plan.md` — Prioritized next steps for Ivo, Pipe, Lu

## Workflow

```
SSP responses (Excel) → Analysis (01) → Audit per account (02) → Fix gaps → Schedule demos (03)
```

The audit (02) is the heavy lift. For each TT-ready account you need to verify in the platform:
- Raw metrics loaded correctly
- Computed metrics / formulas exist and make sense
- Targets created with correct baselines and periods
- Projects linked and in correct phases
- Dictionaries configured (unit conversions, etc.)

Only after the audit passes can you confidently demo to the client.
